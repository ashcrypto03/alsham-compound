# تعطيل عبارة Powered by Chatwoot بالطريقة الرسمية (Self-Hosted)

هذا الملف يشرح **كامل الخطوات العملية** التي تم تنفيذها لتعطيل عبارة  
**Powered by Chatwoot** من ويدجت المحادثة، باستخدام الطريقة الرسمية المدعومة من Chatwoot،
بدون CSS، بدون التلاعب بالواجهة، وبدون التأثير على التحديثات المستقبلية.

تم تطبيق هذه الخطوات على Chatwoot مستضاف ذاتيًا عبر Docker باستخدام Dokploy.

---

## المتطلبات
- Chatwoot مستضاف ذاتيًا (Self-Hosted)
- وصول SSH إلى السيرفر
- Docker يعمل على السيرفر
- صلاحيات لتنفيذ أوامر docker

---

## الخطوة 1: الدخول إلى السيرفر

نبدأ بالدخول إلى السيرفر عبر SSH:

```bash
ssh user@SERVER_IP
```

---

## الخطوة 2: عرض الحاويات العاملة

نستعرض جميع الحاويات لمعرفة اسم حاوية Chatwoot:

```bash
docker ps
```

سيظهر أكثر من حاوية خاصة بـ Chatwoot، عادةً:
- حاوية Rails (التطبيق الرئيسي) ← نستخدمها
- حاوية Sidekiq (مهام خلفية) ← لا نستخدمها

اسم الحاوية المطلوبة يكون مشابهًا لـ:
```
n8nautomaxi-chatwoot-xxxx-chatwoot-rails-1
```

---

## الخطوة 3: الدخول إلى حاوية Chatwoot Rails

بعض صور Chatwoot لا تحتوي على bash، لذلك نستخدم sh:

```bash
docker exec -it <chatwoot_rails_container_name> sh
```

بعد الدخول سنكون غالبًا داخل المسار:
```
/app
```

---

## الخطوة 4: فتح Rails Console

داخل الحاوية ننفّذ:

```bash
bundle exec rails c
```

عند ظهور:
```
irb(main):001:0>
```
فهذا يعني أننا داخل Rails Console بنجاح.

---

## الخطوة 5: معرفة رقم الحساب (Account ID)

تعطيل الشعار يتم على مستوى الحساب (Account)، لذلك يجب معرفة رقم الحساب الصحيح:

```ruby
Account.all.pluck(:id, :name)
```

مثال على الناتج:
```ruby
[[1, "Automaxi"]]
```

في هذه الحالة:
- Account ID = 1

---

## الخطوة 6: تفعيل تعطيل البراندينغ رسميًا

في نسخة Chatwoot المستخدمة، التعطيل يتم عبر خاصية Boolean باسم:

```
feature_disable_branding
```

ننفّذ الأوامر التالية داخل Rails Console:

```ruby
a = Account.find(1)
a.update!(feature_disable_branding: true)
```

إذا لم يظهر أي خطأ، فهذا يعني أن التفعيل تم بنجاح.

---

## الخطوة 7: التأكد من نجاح التفعيل

نقوم بالتحقق بالأمر التالي:

```ruby
Account.find(1).feature_disable_branding?
```

إذا كانت النتيجة:
```ruby
true
```
فالميزة مفعّلة بنجاح.

---

## الخطوة 8: الخروج من الكونسول والحاوية

الخروج من Rails Console:

```ruby
exit
```

ثم الخروج من الحاوية:

```bash
exit
```

---


## النتيجة النهائية

بعد تنفيذ جميع الخطوات:
- تختفي عبارة Powered by Chatwoot من ويدجت المحادثة
- بدون CSS
- بدون كسر التحديثات
- باستخدام ميزة رسمية مدعومة من Chatwoot

---

## ملاحظات مهمة
- التعطيل يتم على مستوى Account
- لا حاجة لإعادة تشغيل الحاويات
- الطريقة آمنة ومستقرة مع التحديثات
- هذه أفضل طريقة لتقديم تجربة White-Label احترافية

---

تم توثيق هذه الخطوات كما نُفذت عمليًا خطوة بخطوة.
